Que es GitHub, Historia y otros Detalles

GitHubPublicado: 11-04-2019 | Actualizado/Corregido: 18-04-2019 | Comunidad: Crear un Post, Eventos Devs, Foro

Nube Colectiva

Post Tags

backend, frontend, github

Dentro de las Plataformas para Desarrolladores podemos mencionar a muchas que cumplen un rol muy destacado en los grandes proyectos que se han y se vienen desarrollando en la actualidad, una de estas plataformas es GitHub que quizs marca un antes y un despus en el mundo del Desarrollo de Software, Web y Mvil, vamos con el artculo.

Veamos varios puntos importantes de GitHub como que es, historia, como usarlo, que proyectos se encuentran alojados en GitHub.

Que es GitHub ?

Es una plataforma en la Nube (Internet) que sirve como almacenamiento de cdigo fuente y archivos de un proyecto de Software, Web, Aplicacin Mvil, etc. todo esto de manera Gratuita, pero GitHub cuenta con su versin de Pago para poder usar otras funciones en su plataforma. Hasta la fecha de este artculo GitHub cuenta con versin de pago para Desarrolladores Independientes o para Empresas (GitHub Enterprise), en el futuro pueden agregar nuevos planes y tarifas, eso depende de la Plataforma.

GitHub te permite trabajar llevando un control de las versiones de tu proyecto, por ejemplo si un da decides subir el cdigo y los archivos de una primera parte de tu proyecto y al siguiente da otra versin y as sucesivamente todos los das, entonces te das cuenta que se estn creando siempre diferentes versiones de tu proyecto, todo se va almacenando en un repositorio que debes de crear para tu proyecto.

Tambin te permite llevar un control de versiones cuando estas trabajando en equipo con otros Desarrolladores, por ejemplo si tu equipo de Desarrollo de un proyecto esta conformado por 3 personas o ms, cada miembro del equipo va ir subiendo su propio avance del proyecto y pueden cometer el error de chancar lo que el otro desarrollador puede haber subido al repositorio, entonces para que no pase esto GitHub lanza un mensaje que nos alerta que estas subiendo un versin obsoleta o que no corresponde.

La plataforma GitHub esta desarrollada con muchas tecnologas, entre ellas esta el Lenguaje de Programacin Ruby por medio del Framework Ruby on Rails, si deseas conocer ms sobre este Framework, te recomendamos leer nuestro artculo Que es Ruby on Rails (RoR) y otros detalles.

Historia

Le genial plataforma GitHub fue creada por Chris Wanstrath , PJ Hyett , Tom Preston-Werner y Scott Chacon ay por Febrero de 2008 en San Francisco – EEUU y hasta la fecha sigue estando ubicada en el mismo lugar.

En Febrero de 2009 los integrantes de la compaa anuncian en una entrevista para Yahoo que GitHub ya haba alcanzado ms de 46 mil repositorios pblicos, una buena cantidad que indicaba la popularidad que se iba a venir para Github.

En Julio de ese mismo ao 2009, GitHub alcanza los ms de 135 mil repositorios.

Ya para el ao 2010 por el mes de Julio, Github contaba con ms de 1 milln de repositorios, el siguiente ao 2011 en el mes de Abril ya contaba con ms de 2 millones de repositorios y as fue creciendo en popularidad mes tras mes, ao tras ao.

Para Junio de 2011 Github la pgina ReadWriteWeb inform que Github haba superado a Source Forge y Google Code, plataformas que en ese momento alojaban grandes cantidades de proyectos.

El 16 de Enero de 2013 Github se afirma notablemente anunciando que haba superado los ms de 10 millones de repositorios en su plataforma.

Github decide abrir su primera oficina fuera de EEUU, esta oficina sera en Japn aya por Junio 2015.

El 28 de Febrero de 2018 Github sufre un ataque DDoS, el segundo mayor ataque en su historia, sufri un pico de trafico de 1.35 terabits por segundo.

El 4 de Junio de 2018 Microsoft anuncia que hizo un acuerdo con Github para comprarlo por 7.5 mil millones de dolares, esta compra se cierra el 26 de Octubre de 2018.

El 19 de Junio de 2018 Github empieza a ofrecer paquetes de educacin gratuita en diferentes escuelas.

Github sigue creciendo constantemente y se ha vuelto la plataforma ms popular para alojar proyectos en la nube, tiene otros competidores como Bit Bucket, Mercurial, GitLab entre otros, pero al parecer Github sigue predominando hasta la fecha de este artculo, en el futuro no sabemos que pueda pasar, lo importante es aprovechar esta plataforma para nuestros proyectos.

Como usar GitHub ?

Para usar GitHub es muy sencillo, solo debes de registrarte en la plataforma GitHub, paso seguido debes de crear un repositorio para tu proyecto siguiendo los pasos que estn en nuestro artculo Como crear un Repositorio Github y subir los archivos de tu Proyecto modo Fcil de esta manera tendrs tus primeros pasos con GitHub.

Tambin puedes Configurar GitHub en la consola de comandos Git para poder trabajar de manera dinmica creando repositorios, controlando versiones de tu proyecto, entre otras tareas en GitHub. Puedes leer nuestro artculo Como configurar GitHub en Git + Clonar y Editar un Repositorio en donde explicamos como Clonar y Editar un repositorio nicamente, pero en otros tutoriales crearemos tareas ms avanzadas con Git.
